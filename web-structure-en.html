Structure Story for Makers

The long conversation about tags, meaning, and reusable document structure

From mainframe line printers to custom elements and public design systems, teams kept asking the same thing: “How do we describe this document so humans and software read it well?” This timeline captures those moments.

Tap any year to open a quick story—who raised the problem, how they solved it, and why later builders kept the pattern. Jargon appears alongside friendly explanations so you can stay in the flow.

Selecting a year opens a dialog close to the button so you can keep your reading position.

1960s

Mainframes learn to paginate

Researchers taught machines to mark sections so printers could format reports automatically. Structure became a programming problem.

1970s

Toward SGML standards

Government archives demanded shared rules for tagging, pushing academics to draft standard generalized markup language.

1980s

Standards meet hypertext

SGML became an international standard just as CERN researchers drafted hypertext notes that evolved into HTML.

1990s

HTML and the DOM find order

IETF and W3C teams cataloged elements and scripting hooks so browsers and editors could treat documents consistently.

2000s

Richer semantics for web apps

Bottom-up communities and new working groups extended HTML to describe data, feeds, and application state.

2010s

HTML5 and accessibility patterns

The HTML standard stabilized while accessibility authorship guides ensured structure and assistive tech worked hand in hand.

2020s

Components and design systems

Custom elements, design tokens, and reference libraries made structural choices repeatable across products.
