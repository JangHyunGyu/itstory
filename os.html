Operating System Story for Builders

운영체제가 하드웨어를 나누고 사람을 돕는 방식이 변해 온 과정

메인프레임을 쉬지 않고 돌리던 배치 모니터, 터미널 앞에서 실시간으로 명령을 주고받게 만든 타임셰어링, 데스크톱과 모바일에서 자연스러운 UI를 만든 그래픽 운영체제까지 이어집니다.

모바일과 오픈소스, 클라우드가 운영체제를 어떻게 다시 설계하게 만들었는지 연도별로 살펴보세요. 버튼을 누르면 당시 엔지니어와 사용자의 고민, 그다음 세대에 남긴 흔적을 모달로 읽을 수 있습니다.

버튼을 선택하면 같은 화면에서 자세한 이야기를 읽을 수 있습니다.

1950s

배치 모니터가 메인프레임을 정리하다

비싼 메인프레임을 쉬지 않게 돌리기 위해 작업 순서를 자동으로 제어하는 소프트웨어가 등장했습니다.

1960s

타임셰어링과 이식성의 시대

서로 다른 기종에서도 같은 소프트웨어를 돌리고, 여러 사용자가 동시에 터미널로 접속하려는 요구가 높아졌습니다.

1970s

마이크로컴퓨터와 가상 메모리

작은 기기에서도 운영체제가 저장장치를 추상화하고 여러 사용자를 지원하기 시작했습니다.

1980s

PC 시대와 그래픽 데스크톱

디스크 기반 운영체제가 표준화되고, 아이콘과 마우스 중심의 인터페이스가 대중화되었습니다.

1990s

네트워크와 오픈소스의 확장

인터넷과 플러그 앤 플레이 요구가 커지면서 운영체제가 공개 협업과 소비자 기능 모두를 품었습니다.

2000s

모바일과 UNIX 계보의 재결합

데스크톱 UNIX가 세련된 UI를 입고, 리눅스 기반 모바일 OS가 대중화의 문을 열었습니다.

2010s

클라우드 네이티브 런타임

컨테이너와 오케스트레이션이 운영체제를 여러 대의 서버를 묶는 제어 평면으로 확장했습니다.

2020s

실리콘 전환과 하이브리드 데스크톱

맞춤형 칩과 클라우드 PC가 운영체제를 로컬과 원격 환경을 아우르는 플랫폼으로 바꾸고 있습니다.

더 깊이 읽을 거리

운영체제가 배치 작업, 타임셰어링, 모바일과 클라우드를 품어 가는 과정을 기록한 1차 자료와 회고를 모았습니다.
