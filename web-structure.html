Structure Story for Makers

태그와 의미, 문서 구조를 붙잡으려는 사람들의 기록

긴 보고서를 자동으로 조판하던 메인프레임 시대부터, 의미 있는 태그와 웹 컴포넌트로 페이지를 나누는 오늘까지. “이 문서를 어떻게 설명할까?”라는 질문이 남긴 발자국을 모았습니다.

연대기 버튼을 누르면 사건을 만든 사람들의 대화와, 구조를 다듬기 위해 어떤 선택을 했는지 들려주는 모달이 펼쳐집니다. 복잡한 용어는 본문 안에서 다시 풀어 드릴게요.

연도 버튼을 누르면 같은 화면에서 자세한 이야기를 읽을 수 있는 대화 상자가 열립니다.

1960s

메인프레임과 문서 자동 조판

대학과 기업 연구소는 기계가 보고서를 자동 조판하도록 실험하며, 문서 구조를 코드로 표현하는 방법을 시험했습니다.

1970s

SGML을 향한 표준 논의

학회와 정부 기관이 문서 교환 규칙을 만들자며 모였고, 태그로 구조를 설명하는 방식을 검사했습니다.

1980s

표준 문서와 하이퍼텍스트의 만남

SGML 표준이 확정되고, CERN 연구원들이 하이퍼텍스트 문서를 제안하며 웹 태그의 씨앗이 뿌려졌습니다.

1990s

HTML과 DOM, 의미를 정리하다

IETF와 W3C가 HTML 요소와 문서 객체 모델을 정리하며, 웹 문서를 프로그램이 읽을 수 있게 만들었습니다.

2000s

의미 확장과 웹 애플리케이션 준비

WHATWG가 HTML5 초안을 쓰고, 마이크로포맷이 구조를 풍부하게 하자 웹 문서는 응용 프로그램의 기반이 되기 시작했습니다.

2010s

HTML5 완성과 접근성 패턴

웹 표준이 완성되면서 시맨틱 요소와 WAI-ARIA 패턴이 자리 잡아, 구조와 접근성이 함께 다뤄졌습니다.

2020s

컴포넌트와 디자인 시스템

커스텀 엘리먼트와 디자인 시스템 가이드가 보편화하며, 구조를 코드로 재사용하는 흐름이 굳어졌습니다.

더 깊이 읽을 거리
